Models That Can Be Chosen for Simulation

Prepared for the simulation, models present different types of behavior, such as: self-driven behavior or combined self-driven and external input driven behavior. Behavior may also differ by number of independent processes or demonstrate chaotic but yet deterministic character behavior in which the entire behavior of the system is viewed as a single process identifying the independent sub-processes in which is merely impossible.

All the models in this presentation are intentionally of limited size since the purpose of the demo is not to show how big a qualitative dynamical model can be, but to demonstrate the principle of transformation of the current state of a model into its next state. The models are organized in three groups.

First group of models: "Full State Situations and Self Driven Models" consists of two model. Each possible combination of all states of the state variables of the models of this group (presented as the State Vector of the equation) characterises a particular situation that may occur in the modeled system.

Model 1.1: demonstrates case when each state of the model is recognized as the situation to which the model responds by transitioning to the next state, corresponding to the situation.

Model 1.2: is a variation of Model 1.1 in which only some states of the model are recognised as situations to which the model should respond, but the others are not.

Second group: "Partial Situations and Input Driven Models" consists of one model:

Model 2.1: demonstrates case where all state variables of the model are splitted into two groups that may change independently. The model behavior, thus, is constituted of two independent processes. In order to present this clearly, one process is finite while the other is endless.

Third group: ""Partial Situations and Mixed Input Driven Models" consists of one model:

Model 3.1: demonstrates case where there are no explicitly separated processes. Instead, there is only one process in which unpredictably occurring partial situations are the causes of fragmentary, e.g. partial, changes in the full state of the model.

For Those Who Want to Know More

Observed behavior of qualitative systems is perceived as a process of occurrence of situations in the system and its reactions to these situations, expressed in the execution of actions that change state of some properties of the system. Therefore, the task of the model, reproducing the behavior of a qualitative system, is to systematically recognize the situations represented by combinations of the states of groups of properties that have arisen as a result of the previous behavior of the system and to make changes in the current state of the system corresponding to the recognized situations.

Presented on this page, solution to modeling the behavior of qualitative systems is given in the form of a vector state equation describing the process of computation of the correspondences "Situation -> Change of the State of Properties" as a composition of vector operations and matrix transformations of the current state of the system into its next state. The structure of the state equation and description of the process of its computation are in the sections to follow.

Parts of the Qualitative Dynamical System Equation

The equation consists of eight elements listed in the order of their invocation in the computation, i.e. from right to left.

  1. External Input Vector has ___ background and is an optional element of the model. It is included in the model in case when modeling technique requires the model simulation process to be driven by external input and is omitted otherwise.
  2. Current State Vector has ___ background and is the representation of the state of the modeled system at the moment t, i.e. at the time when the iteration of simulation is going to begin.
  3. Recognition Matrix has ___ background and is the Current State Vector transformation operator. The number of columns of this matrix corresponds to the number of elements of the Current State Vector of the system, whereas the number of rows is determined by the number of situations that are supposed to be recognized and which, for that purpose, have to be included in the matrix.
  4. Recognized Situations Vector has ___ background and is the result of transformation of the Current State Vector of the System. It holds symbols indicating facts of recognition of presence of expected situations in the Current State Vector of the system.
  5. Generation Matrix has ___ background and is another transformation operator. It is applied to the Recognized Situations Vector and generates values of the Proposed State Vector. Each column of this matrix, thus, is associated with the particular recognized situation, and each row with the particular element of the Proposed State Vector.
  6. Proposed State Vector has ___ background and is the result os Generation transformation. it is a collection of changes that should be infused in the Current State Vector. As at the moment of computation of iteration some of expected situations may exist while some others might not, the Proposed State Vector may be partially defined. Some of its elements may hold symbols denoting the valid state of the properties of the system, whereas others may appear to be just the auxiliary symbols used in the process of computation indicating that expected situation was not recognised.
  7. Dobbed State Vector has ___ background and is a clone of the Current State Vector. It is presented here for just convenience to have one more view of Current State Vector right in the place where it and the Proposed State Vector are used as the arguments of the vector operation computing content of the Next State Vector of the System.
  8. Next State Vector has ___ background and is the result of infusion of the Proposed State Vector into the Dobbed State Vector. This is temporary vector that is needed to present the computed result of an iteration - values of the Current State Vector at the time t+i, where i is the index of the completed iteration. Values of the elements of this vector are copied to the corresponding elements of the Current State Vector as soon as a step button is clicked and the simulation process proceeds to the initial state of the next iteration.

Representation of Values of the Elements of the Parts of the Equation

It is assumed that all values of state variables, which in the state-space representation of the model are defined in the form of the elements of the Current State Vector, are qualitative and therefore, can be denoted by symbols taken from a certain set of symbols. On this page the alphabet of the Latin characters is used.

There are two sets of symbols that are used to represent state of the model under simulation: a set of domain symbols, and a set of auxiliary symbols. The set of domain symbols D is defined as:

D = { A, B, C, …, Z, … },

and the set of auxiliary symbols X, defined as:

X = { -, !, $, # }.

Symbols of set D are used to denote values of elements of the model representing states of the properties of the modeled system. And the symbols of set X are used to denote values of those elements of state of the model which represent the state of the process of computation of the equation. Meaning of the symbols of the set X is the following:

Symbol “-” is a kind of “zero” of a model and is used to represent the values of those elements of the vectors and the matrices of the equation which should be excluded from the process of computation.

Symbol “!” is used to denote one of two possible results of the procedure of recognition of a situation (see the definition of the Equivalence and Conjunction operations in the Appendix "Algebra of Symbol"). It is yielded to present a fact asserting that a particular expected value presented in a particular element of a particular row of the Recognition matrix has compared to the corresponding element of the Current State Vector, and thus the compared element of the Current State Vector is considered as the recognized element of the expected situation.

Symbol “$” is used to denote second possible results of the recognition of a situation. It is an alternative to the symbol “!” and serves for presenting the fact asserting that a particular expected value presented in a particular element of a particular row of the Recognition matrix was not compared to the corresponding element of the Current State Vector, and thus the entire expected situation presented in the processed row of the matrix cannot be considered as recognized.

Symbol “#” is to denote a particular result of the Generation transformation (see the definition of the Production and Disjunction operations in the Appendix “Algebra of Symbol"). It is yielded by the Disjunction operation to eliminate a conflict that occurs when two, or more, situations are simultaneously recognized by the Recognition matrix and then transformed by the Generation matrix to different proposed values.

Initialization of the Vectors and the Matrices

In order for the equation to be computed its elements should be initialized. The initialization is conducted automatically right after the model is picked up from the Model Chooser dropdown list.

Initialization of vectors. The Current State Vector is initialized with the model initial state provided by specification of the model. All other vectors are populated with the symbol “-” indicating that their values are not yet computed.

Initialization of matrices. Based on the mechanism of transformation of matrix operator input vector into its output vector implemented as the procedure of systematic multiplication of input vector by each row of the matrix, principle of initialization of matrices is determined by the role of the matrix and is applied to all rows of the initialized matrix. Accordingly:

(1) Initialization of the Recognition Matrix. The matrix is initialized in such a way that its every row is filled with the expected combination of states of the group of properties of the modeled system that characterizes the situation to which the system (and the model) should respond by changing state of one or more of the properties of the system.

(2) Initialization of the Generation Matrix. The matrix is filled with the to be generated symbols so that each of the symbols to be placed at the intersection of the row corresponding to the element of the Proposed State Vector and of the column corresponding to the situation, in case of arising of which this symbol is assigned to the element of the Proposed State Vector as the value proposed as the state of the element of the Next State Vector of the System.

Highlighting the Active Vector Background and the Changes in
the Status of Their Elements

As the simulation process is a sequence of computational steps of execution of vector or matrix transformations, there is always a vector toward which a currently executed operation is targeted. This vector is considered by the simulation process as an active one. In order to emphasize the active status of a vector, the background of the currently active vector is highlighted by the simulation process in the pink color.

When carrying out a simulation process, the Simulation Engine presents changes in the state of the elements of the resulting vector of every executed operation in the view of the model on the page. For this purpose each element of the vector which changed its state as a result of an operation is highlighted with the color corresponding to the color of the current (i.e. the usual or the active) background of the vector.

Qualitative Systems, Their Behavior, Modeling Technique and
the Behavior Simulation Process

Imagine a discrete action device, the state of which is characterized by a certain collection of changing over time, discrete properties. When talking about the state or behavior of these devices, we often merely list these properties in the form of affirmative statements giving them, as they say, qualitative evaluations. So, for example, we may say: “My car is red”, “The Earth is orbiting the Sun” or "The weather today is pretty nice". Therefore, following the adopted terminology the collections of properties depending one on another, and the devices themselves, may be naturally classified and referred to as the Qualitative Systems or the Qualitative Dynamical Systems. Characteristics of the qualitative systems behavior, their modeling technique and the behavior simulation procedure are:

(a) The behavior of a qualitative system - is a process of discrete changes in its state over time; (b) The qualitative system modeling technique - is the representation of these systems in the form of space-state equation and the description of a way of computing a particular change in the system by describing the procedure for computing the next state of the system as a function of its current state; and (c) The behavior simulation procedure - is an iterative process of reproducing the sequences of discrete changes in the state of the system, which begins from the first iteration of computation of the state equation, based on the initial state of the model, and then is being carried out through the repeated computations of the equation, in which the next state of the system, obtained at the just performed iteration, is used as its current state at the next iteration.

Since the process of computation of the space-State equations requires different manipulations with the vectors in the symbolic space states, a number of algebraical operations with the symbols are needed for accomplishing the transformations needed. These operations defined on the set of symbols were discovered and are described in the next section below.

The Algebra of Symbols

Described in the previous sections, vector equations are the mathematical models of the behavior of qualitative dynamical systems, and the vector operations providing the computation of these equations are the mathematical operations on the qualitative vectors i.e. vectors that are the elements of the space of qualitative states of qualitative dynamical systems. Therefore, just as the operations with vectors of a numerical space that are implemented as a composition of operations defined on a set whose elements are used to represent numerical characteristics of the system and, thus, are the numbers, the operations on vectors of qualitative space have to be implemented as a composition of operations defined on a set whose elements are used for representation of qualitative states of properties; that is, in the given case, on the set the elements of which are the symbols.

To satisfy the requirements of the vector operations described earlier, the Algebra of Symbols is defined on a combined set of symbols, defined as the union of a set of domain symbols D and a set of auxiliary symbols X. This allows to implement the computation of vector equations as homogeneous (with no branches) process in which the conditional choices affecting the result of the computation are made inside the operations as the static part of the definition of the operations, i.e. at the level of mapping the arguments of the operation to its result.

The feasibility of constructing such a system of operations was explored in the framework of the project described on this page and an algebraic system providing operations necessary for implementation of operations on vectors in the qualitative space, called Algebra of Symbols, was defined. The Algebra defines five operations necessary for computation of the qualitative vector state equation. They are: Application, Equivalence, Conjunction, Production and Disjunction.

More details about the Algebra of Symbol can be found on the page "Algebra of Symbol: Definition, Explanation and Application".

Qualitative Dynamical System Equation Computation

The process of computation of the equation consists in performing a sequence of five operations that transform the content of the current state vector of the system into its next state. These operations are executed as five successive steps which are carried out from right to left.

The first, optional, step of computation of the equation is involved in the case when the behavior of the system is determined by external influences on it. It consists in performing the "Application" (<) operation, which "injects" the values of the vector of external influences into the vector of the current state of the system. This procedure is expressed in replacing the values of the elements of the current state vector with the values of non-empty elements of the vector of external influences. The result of the operation is the modified current state vector.

The role of the second step is to multiply the current state vector by the Recognition matrix. This step is performed as a procedure of 1) row-by-row comparison of the values of the elements of the rows of the Recognition matrix with the values of the corresponding elements of the current state vector and 2) further, following the comparisons, the computation of the n-ary conjunction of the result of the comparisons, where "n" represents the size of the current state vector and also the number of columns of recognition matrix. The result of this transformation is the Recognized Situations Vector whose elements contain either the symbol "!" meaning that all elements of the row of the matrix associated with the element of the vector of recognized situations have been compared and thus the expected situation described in the row of the matrix is considered recognized, or, alternatively, the symbol "$" which means that at least one of the values of the current state vector has not compared with its expected value and therefore the situation is deemed unrecognized.

The third step of the computation, is, like the second one, the multiplication of a vector by a matrix. But now the transformation consists in multiplication of the just calculated vector of recognized situations by the Generation matrix. The purpose of this transformation is to map the facts of recognition of the situations, presented as symbols "!" in the elements of recognized situations vector into the values of the vector of the proposed next state of the system. Procedure of such state-generating transformation consists in the element-by-element computation of the values of the vector of proposed next state of the system and, correspondingly, is implemented as 1) production for each element of the recognized situations vector, values located in the row of the generation matrix corresponding to that computed element and thus finding all the values corresponding to all recognized situations, and 2) the choosing out of all the produced values the only one which is going to be used as the value of the element of the vector of proposed next state of the system.
